Criando uma ramificação de trabalho

Em Git, uma ramificação (ou branch) é uma linha de desenvolvimento independente que permite que você trabalhe em um conjunto de alterações sem afetar o código principal.

Cria uma branch e já muda para a mesma.

git checkout -b develop/natorjunior

Agora é a hora de fazer as suas alterações.

Em seguida, você pode usar o comando git status para verificar as mudanças feitas localmente.

git status

Logo em seguida voce pode adicionar os arquivos que voce modificou, para em seguida fazer o commit;

git add diretorio_teste/meu_programa.py
git add diretorio_teste/plotly_dash.py
git add diretorio_teste/config.py

Agora vamos fazer o commit dos arquivos alterados.

Faça o commit das alterações e inclua uma mensagem;

git commit -m "Meus primeiros programas"

Pronto, as alterações foram finalizadas, agora é só subir para o github

Faremos o push das alterações.

git push --set-upstream origin develop/natorjunior

Comandos básicos do git

Clona um repositório remoto existente para o seu computador.

git clone url-do-repositorio-remoto

Adiciona um arquivo específico ao próximo commit.

git add <arquivo>

Registra as alterações adicionadas no repositório com uma mensagem descritiva.

git commit -m "mensagem"

Exibe o estado atual do seu repositório, mostrando arquivos modificados, adicionados ou removidos.

git status

Lista todas as branches existentes no repositório.

git branch

Muda para a branch especificada.

git checkout <branch>

Obtém as últimas alterações de um repositório remoto e as mescla com o seu repositório local.

git pull

Envia as alterações locais para um repositório remoto.

git push

Incorpora as alterações da branch especificada à branch atual.

git merge <branch>
